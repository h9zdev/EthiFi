## рҹ§ӯ What is EthiFi?

**EthiFi** is a low-cost, open-source WiFi pentesting platform built on the **ESP8266** microcontroller. Designed for security researchers, students, and enthusiasts who want to learn about wireless network vulnerabilities вҖ” without spending a fortune.

> вҡ пёҸ **Ethical Use Only** вҖ” EthiFi is intended strictly for **authorized testing and educational purposes**. Never use on networks you do not own or have explicit permission to test. Unauthorized access to networks is illegal.

## рҹ’ҫ Flash Size

> вҡ пёҸ **The flash size is an important factor!** Choosing the wrong flash size setting will cause the firmware to fail or behave unexpectedly.

Make sure to select the correct flash size for your specific ESP8266 module before flashing. Different modules ship with different flash chip sizes вҖ” check your board's specs carefully.

![ESP8266 Module Flash Sizes](https://raw.githubusercontent.com/wiki/spacehuhn/esp8266_deauther/img/esp_modules.jpg)

| рҹ“Ұ Module | рҹ’ҫ Flash Size |
|---|---|
| ESP-01 (old) | 512 KB |
| ESP-01 (new) | 1 MB |
| ESP-07 | 1 MB |
| ESP-12E / NodeMCU | 4 MB |
| Wemos D1 Mini | 4 MB |

> рҹ’Ў When in doubt, try **4MB** вҖ” most modern NodeMCU and D1 Mini boards ship with 4MB flash.

## рҹ”§ Driver Installation

Most ESP8266 boards use one of two USB-to-Serial chips: **CP2102** or **CH340**. If your board isn't detected when plugged in, you need to install the correct driver.

![NodeMCU Serial Module Types](https://raw.githubusercontent.com/wiki/spacehuhn/esp8266_deauther/img/nodemcu_serial_modules.jpg)

> рҹ’Ў **How to tell which chip you have:** Look at the small IC near the USB port on your board. It will be labeled `CP2102` or `CH340G`.



### рҹ”ө CP2102 Driver *(Silicon Labs)*

Used on many NodeMCU v1, DSTIKE boards, and others.

| рҹ–ҘпёҸ OS | рҹ“Ҙ Download |
|---|---|
| Windows / macOS | [Silicon Labs вҖ” CP210x VCP Drivers](https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers) |
| Linux | вң… Built-in вҖ” no install needed |

**Steps (Windows):**
1. рҹ“Ҙ Download the driver `.zip` from the Silicon Labs link above
2. рҹ“Ӯ Extract and run `CP210xVCPInstaller_x64.exe` (use `x86` for 32-bit Windows)
3. вң… Follow the installer prompts and complete installation
4. рҹ”Ң Unplug and re-plug your board
5. Open **Device Manager вҶ’ Ports (COM & LPT)** вҖ” you should see:
   `Silicon Labs CP210x USB to UART Bridge (COMx)`



### рҹҹў CH340 / CH341 Driver *(WCH)*

Used on NodeMCU v3 (Lolin), Wemos D1 Mini, and many cheap clone boards.

| рҹ–ҘпёҸ OS | рҹ“Ҙ Download |
|---|---|
| Windows | [WCH вҖ” CH341SER.EXE](https://www.wch-ic.com/downloads/CH341SER_EXE.html) |
| macOS | [WCH вҖ” CH341SER MAC](https://www.wch-ic.com/downloads/CH341SER_MAC_ZIP.html) |
| Linux | вң… Built-in (kernel 2.6+) вҖ” no install needed |

**Steps (Windows):**
1. рҹ“Ҙ Download `CH341SER.EXE` from the WCH link above
2. в–¶пёҸ Run the installer and click **INSTALL**
3. вң… Confirm the success message
4. рҹ”Ң Unplug and re-plug your board
5. Open **Device Manager вҶ’ Ports (COM & LPT)** вҖ” you should see:
   `USB-SERIAL CH340 (COMx)`

## рҹ©ә Troubleshooting

### вқҢ Board not detected / No COM port shown

- Install the correct driver вҖ” [CP2102](#-cp2102-driver-silicon-labs) or [CH340](#-ch340--ch341-driver-wch)
- Try a different USB cable вҖ” many cables are **charge-only** and carry no data
- Try a different USB port on your computer
- On Windows: open **Device Manager** and look for unknown devices with вҡ пёҸ icons


### вқҢ Flash fails or board disconnects mid-flash

- Hold the **FLASH / BOOT** button on your board while initiating the flash
- Lower the baud rate in your flasher (try `115200` instead of `460800`)
- Make sure no other program (e.g. serial monitor) is using the COM port
- Try a shorter USB cable


### вқҢ Wrong flash size selected

- If firmware crashes or the web UI doesn't load, you likely selected the wrong flash size
- Re-flash with the correct size вҖ” see the [Flash Size](#-flash-size) section above



### вқҢ Web UI not loading at `192.168.4.1`

- Make sure you are connected to the **EthiFi** WiFi, not your home router
- Disable mobile data on your phone (it may silently switch away from EthiFi)
- Use `http://` explicitly вҖ” not `https://`
- Try a different browser or clear your cache



### вқҢ Arduino IDE doesn't show Deauther boards

- Double-check the board manager URL was added correctly in **File вҶ’ Preferences**
- Restart Arduino IDE after adding the URL
- Make sure you clicked **Install** in Boards Manager after searching



### вқҢ Upload fails in Arduino IDE

- Select the correct **COM port** under **Tools вҶ’ Port**
- Make sure no other app is using the port (close any open serial monitors)
- Press and hold the **BOOT** button on your board during upload, release when upload starts
